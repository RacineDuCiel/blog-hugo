Chapter 10

#certifications/CASP

Overall Context: Technology is advancing rapidly, increasing connectivity (including critical infrastructure) and blurring boundaries. CASP+ professionals need ongoing awareness, situational analysis, and the ability to evaluate and integrate new technologies securely, considering performance, cost, and risk. This chapter focuses on analysis and impact. Objective 1.8: Impact of Emerging Technologies on Enterprise Security & Privacy A. General Considerations

  • Impact: New tech brings new threats, governance challenges, and opportunities for attackers. Can be disruptive.
  • Governance: Frameworks must be agile to keep up with rapid tech changes and provide policy/regulation guidance.
  • Accessibility: Easy access/adoption can enable malicious use.

⠀B. Specific Emerging Technologies 1 Artificial Intelligence (AI): * Concept: Systems problem-solving and learning by analyzing input/data to simulate human intelligence. * Examples: Deloitte CortexAI (autonomous enterprise functions), Vaccine development research. Hardware optimization ongoing. 2 Machine Learning (ML): * Concept: Component of AI. Uses algorithms and large datasets to make predictions based on a learning model (without explicit rules). * Security Use: Identifying malware (heuristic, signature, etc.), predicting search queries. 3 Deep Learning: * Concept: Advanced ML mimicking human brain’s neural network structure. Solves complex problems, finds hidden patterns in large, tagged datasets. High accuracy focus. * Natural Language Processing (NLP): Deep learning + linguistics. Enables computer understanding of human language (spoken/written). Examples: GPS, digital assistants, chatbots, Google Translate (imperfect). * Deep Fakes: Deep learning used to replace a person’s likeness in images/video. Risk: Disinformation (e.g., fake leader statements), impersonation. 4 Quantum Computing: * Concept: Uses quantum mechanics for computation/communication. Still evolving. * Security Impact: Potential to break classical crypto algorithms relying on factoring large primes (e.g., RSA). Quantum supremacy achieved (Jiuzhang example). * Synergy: Can enhance ML models with larger datasets; Deep Learning aids quantum research. 5 Blockchain: * Concept: Digitized, decentralized, public ledger (often for cryptocurrency transactions). Append-only blocks in chronological order. Distributed Ledger Technology (DLT) implementation. * Function: Verifies transactions without central authority via distributed consensus (mining). Each node gets a copy. * Benefits: Enhanced security (immutable record), transparency, traceability, speed, reduced costs. * Use Cases: Cryptocurrency (Bitcoin), verifying authenticity of any digitized document/record. 6 Homomorphic Encryption (HE): * Concept: Allows computation on encrypted data, yielding an encrypted result that matches computation on plaintext, preserving privacy. * Subfields/Applications: * Private Information Retrieval (PIR): Retrieve item from DB without revealing what was retrieved. Protects querier’s privacy. * Secure Function Evaluation (SFE): Entities evaluate a function using private inputs without revealing the inputs (e.g., “Millionaire Problem” - who is richer?). * Private Function Evaluation (PFE): Evaluate one party’s private data using another party’s private function/software without revealing either. * Strong PIR (SPIR): PIR + ensures querier learns only the queried element (protects DB owner’s privacy too). * Secure Multiparty Computation (MPC): Multiple entities jointly compute a function on private inputs without revealing inputs. Relies on cooperation; aborts if cheating detected. 7 Distributed Consensus: * Concept: Processes in a distributed system agreeing on specific data values during computation. Handles failures/errors. * Examples: Time synchronization, load balancing, aircraft control systems. 8 Big Data: * Concept: Extremely large datasets from diverse sources (Internet transactions, databases, mobile devices, IoT, social media, logs). Requires special tools/databases for analysis. * Tools: Splunk (log aggregation, evolved into big data analysis tool with Search Processing Language - SPL). 9 Virtual/Augmented Reality (VR/AR): * VR: Immersive simulated experience (headsets like Samsung Gear, Oculus). * AR: Digital content overlaid onto the real world (e.g., Pokémon GO, advertising). * Industry Uses: Gaming, Healthcare (surgical training, therapy), Military (simulations), Automotive (design review). * Security/Privacy Concerns: Vulnerable hardware/apps (payment/registration data), hacked live streams (surgery), malware in apps/platforms, data leakage. 10 3D Printing: * Concept: Additive manufacturing; building objects layer-by-layer from digital models using various materials (plastics, metals, ceramics, food). * Benefits: Rapid prototyping, custom parts (medical, aerospace, auto). * Cyber Threats: Intellectual Property (IP) theft (cloud designs, reverse engineering), Sabotage (modifying designs leading to part failure). Regulation/quality control challenges. 11 Passwordless Authentication: * Concept: Logging in without traditional passwords. Relies on other factors: Something you have (phone, token) or something you are (biometrics). * Challenges: Cost, user acceptance/skepticism. * Example: Disney World fingerprint ticket validation (suspended during pandemic). 12 Nanotechnology: * Concept: Science/technology at the nanoscale (billionths of a meter). * Applications: Crop fertilization, medical treatments, energy production. * Concerns: Potential health risks (DNA damage, crossing blood-brain barrier), diagnostic errors (“doctor on a chip”). Safety/security questions. 13 Biometric Impersonation & Systems: * Process: Enrollment (capture sample) -> Authentication Request (compare live sample to template) -> Decision (match/no match). * Accuracy Metrics: * Type 1 Error / False Rejection Rate (FRR): Authorized user denied. * Type 2 Error / False Acceptance Rate (FAR): Unauthorized user accepted. * Crossover Error Rate (CER): Point where FRR = FAR. Lower CER = more accurate system. * System Types: Fingerprint, Hand Geometry, Palm Scan, Retina Pattern (accurate, intrusive, changes over time), Iris Recognition (accurate, less intrusive), Voice Recognition (vulnerable to recording), Keyboard Dynamics. * Considerations: User Acceptance (intrusiveness), Usability (processing time), Impersonation Risk (voice recording). * Data Management: Risk of storing sensitive biometric data in databases. Need secure decommissioning processes.

⠀Objective 3.1: Secure Enterprise Mobility Configurations A. General Management & Configuration 1 Managed Configurations: Centralized control over mobile device settings using profiles/policies. Requires CASP+ guidance & collaboration. 2 Application Control: * Application Wrapping: Adding management layer (policy enforcement, data sharing limits) around an app without modifying the app itself. * Application Permissions: Controlling app access to device resources (camera, mic, contacts). Grant only if trusted/needed. * Mobile Application Management (MAM): Software for IT control over enterprise apps (corporate/personal devices). Enforces policies, separates corporate/personal data, schedules updates. Needs clear policy on employee separation (remote wipe risk). 3 Authentication: * Password/Passcode Policy: Complex (>=6 chars, avoid dictionary/repeats), use Password Managers. * Multifactor Authentication (MFA): Require >1 factor type (know, have, are). True MFA vs. 2-Step Verification. * Token-Based Access: Using physical/software tokens for authentication. Limit failed attempts (lockout). 4 Updates & Patching: * Patch Repository: Central source for managing/viewing mobile patches/baselines. Policy should align with vendor recommendations. * Firmware Over-the-Air (OTA): Remote delivery of OS/firmware updates. Encrypted using device-specific key, stored securely to prevent cloning/downgrades. 5 Remote Wipe: Erasing device data remotely (e.g., upon loss, theft, employee termination). Mitigates unauthorized use risk. Potential for accidental data loss. 6 Wireless Security (WiFi): * WPA3: Current standard. Uses 128-bit AES (or 192-bit mode). Requires Protected Management Frames (PMFs). Better than WEP (broken), WPA2 (insecure). * WPA3 192-bit Mode Crypto: EAP-TLS (ECDH/ECDSA P-384), GCMP-256 (encryption), HMAC-SHA384 (key derivation), BIP-GMAC-256 (management frame protection). * WIDS/WIPS: Detect/prevent rogue APs. 7 Certificates & Management: * Purpose: Secure email, encryption, authentication (Wi-Fi, VPN). * Deployment: MDM/EMM solutions streamline provisioning. * SCEP (Simple Certificate Enrollment Protocol): Scalable but has cryptographic weaknesses (auth methods), risky in open environments like BYOD (unauthorized request risk - VU#971035). 8 Configuration Profiles: Group of settings deployed remotely to manage devices consistently. Contains Payloads (subsets of settings, e.g., mail, security restrictions). 9 Short-Range Wireless: * Bluetooth: Short-range communication. Vulnerable. Version 5.0 (2 Mbps), LE Power Control/Audio (faster pairing, battery life). Used for tethering (slow), peripherals. * Near-Field Communication (NFC): Very short range (<4cm), low data rate (106-424 Kbps). Uses inductance. Used for data exchange (photos), mobile payments (“tap to pay”). Attack risk lower due to proximity. Disable when not needed. 10 Mobile Payments: * NFC-Enabled: Uses NFC coils/inductance. * Mobile Wallets: Apps for managing/making mobile payments. * Peripheral-Enabled: Using readers (e.g., EMV chip readers) connected to mobile devices. 11 Peripherals Control: Restrict USB, Bluetooth, NFC, ports to mitigate data exfiltration risk. Includes smartphones, wearables, tablets when connected. (Also consider work-life balance aspect). 12 Location-Based Controls & Services: * Geotagging: Embedding location data in files (photos). Default setting risk. Used by attackers (Creepy app, conflict zones). * Geofencing: Triggering actions based on device crossing a defined boundary. * Location Services: Apps using device location (GPS). Privacy Risk: Tracking user behavior/preferences, identity theft, unwarranted surveillance (law enforcement). Review privacy settings. 13 Full Device Encryption (FDE): Encrypts entire device storage (OS, files). Protects data if device lost/stolen. Examples: EFS, BitLocker. Mandatory policy item. Takes time, requires password management. 14 Tethering: Sharing one device’s network connection (e.g., cellular) with others via Wi-Fi hotspot, Bluetooth, or USB. * Security Concern: Connects potentially unmanaged devices to network. Unauthorized activation risk. Monitor for unauthorized use. * Carrier Issues: Abuse of “limitless” data plans. Carriers may detect/charge/restrict tethering (“fair use” policy). 15 Airplane Mode: Disables cellular, Wi-Fi, Bluetooth, GPS. Required on flights (FCC). Protects against some attacks/tracking. Conserves battery. 16 DNS Security: * DNS over HTTPS (DoH): Encrypts DNS queries using HTTPS. Increases privacy/security. * Custom DNS: Manually specifying DNS resolver IP addresses (faster/reliable servers, content filtering, ad blocking).

⠀B. Deployment Scenarios 1 COBO (Corporate-Owned, Business Only): Company owns/dictates device and strictly controls content/use. Most secure, least flexible. 2 COPE (Corporate-Owned, Personally Enabled): Company owns device, allows some personal use (apps, browsing). More freedom than COBO, security concerns increase. 3 CYOD (Choose Your Own Device): Company offers limited choice of approved devices. Compromise between user preference (OS loyalty) and manageability. Doesn’t guarantee app freedom. 4 BYOD (Bring Your Own Device): Employees use personal devices for work. Most flexible, highest risk. Requires focus on access control and network security, not device management. (~60-70% adoption).

⠀C. Mobile Device Security Considerations 1 Emergent Threats & Attacks: * Smartphone Malware: Slower adoption than PC malware (bandwidth limits, market fragmentation, shorter lifecycle) but growing as data capabilities increase. Vectors: Malicious sites, email attachments, social media links. * Botnets: Using compromised mobile devices (zombies) for spam, DDoS, crypto mining, etc. * Scareware: Fake/malicious software sold via deceptive marketing (fear tactics). * Money Mules: Individuals transferring stolen funds/goods. Used by botnets (Zeus example). * Smishing: Phishing via SMS messages. * Search Engine Poisoning: Manipulating search results to lead users to malicious sites. * Crimeware Kits: Toolkits for launching attacks (phishing, malware) without technical expertise. * Clickjacking: Hiding malicious links/actions behind legitimate-looking elements. 2 Specific Considerations: * Unauthorized Remote Activation/Deactivation: Maliciously enabling/disabling features (camera, mic) or wiping device. Mitigation: Update OS, avoid unknown apps. * Encrypted/Unencrypted Communications: Use built-in encryption, secure protocols (HTTPS, VPN, WPA3). Encryption process takes time, requires password management. Bluetooth encryption vulnerable. * Physical Reconnaissance: Tracking location via GPS, call logs, SMS records (even deleted), social media geotagging, fitness trackers. Google tracking persists even if history off. Military base locations leaked via fitness trackers. * Personal Data Theft: Via compromised apps, spyware. Includes call logs, SMS, photos, videos, contacts, browsing history. * Unauthorized Application Stores: Bypassing official stores (Apple App Store, Google Play) increases risk of malicious apps. * Jailbreaking (iOS) / Rooting (Android): Removing OS limitations. Allows unapproved software, custom configurations. Voids warranty, increases security risk. Scan devices for this. * Sideloading: Installing apps outside official distribution channels. Bypasses security vetting. High risk. * Containerization (Mobile Context): Isolating work apps/data from personal apps/data on a single device. Enhances security in BYOD/COPE. (Related to Docker concept in Ch 8, but applied to mobile OS). * OEM & Carrier Differences: Android ecosystem fragmentation (multiple OEMs - LG, Samsung; many carriers) leads to customization, bloatware, inconsistent updates. iOS (Apple) is a closed ecosystem (better app/accessory ecosystem, better privacy controls like tracking notifications). Both track user data. * Supply Chain Issues: Risks associated with hardware/software components from various suppliers throughout the device lifecycle. Requires enterprise resilience planning beyond traditional IT BCDR. * eFuse: Hardware chip feature allowing reprogramming/feature alteration in real-time. Used by vendors (Microsoft Xbox, Motorola phones) for anti-tampering (disabling jailbroken/modded devices).

⠀D. Wearable Device Implications

  • Fitness Devices (Fitbit): Risk of location disclosure (GPS tracking) via social sharing. Physical reconnaissance risk (military base example).
  • Watches (Smartwatches): Contain/access sensitive data (location, contacts, email, calls, texts). Personal data theft risk if compromised. Loss/theft concern.
  • Cameras/Glasses: Privacy concerns (recording others). Location data leakage via EXIF metadata. Risk of device transmitting data unknowingly. Augmented Reality glasses (Google Glass) add physical safety risks (distraction).
  • Headsets: Microphone compromise risk. Bluetooth vulnerabilities (encryption bypass, jamming).

⠀E. Health Privacy

  • Connected Medical Devices: Increased connectivity often lacks corresponding security (authentication, input validation). Devices vulnerable to exploitation.
  • HIPAA: US regulation protecting sensitive patient health information (PHI). Privacy is paramount concern.

⠀Objective 3.3: Security for Specific Sectors & Operational Technologies (OT) A. Embedded Systems & IoT 1 Definition: Hardware/software combination for specific purpose, often inside other devices. 2 IoT (Internet of Things): Network connectivity for everyday objects (thermostats, appliances, cameras, sensors). 3 Core Risk: Increased connectivity without adequate security controls (authentication, validation). Devices often “wide open.” 4 Examples & Risks: * Refrigerator Botnet: Compromised smart appliances used for DDoS/spam. * Building Automation: Control mechanical, security, fire, lighting, HVAC. * IP Video: Surveillance cameras, baby monitors. Vulnerable firmware allows unauthorized viewing (Foscam example). * HVAC Controllers: Remote control risks. * Sensors: Home automation (lights, doors), medical devices (pacemakers), vehicle systems. * Physical Access Control: Bluetooth/RFID locks. * A/V Systems: Home/corporate AV networks. * Scientific/Industrial Equipment: (See ICS/SCADA). 5 Hardware Components: * SoC (System on a Chip): Microchip containing all necessary computer components (CPU, GPU, etc.). Example: Raspberry Pi (Broadcom chip). Often pairs FPGA + processor. * ASIC (Application-Specific Integrated Circuit): Chip designed for a single purpose (voice recorder, satellite, phone). * FPGA (Field-Programmable Gate Array): Circuit designed to be configured by customer after manufacturing.

⠀B. Industrial Control Systems (ICS) / SCADA 1 Definition: Systems managing critical infrastructure (pipelines, power grids, manufacturing) and building automation (HVAC, elevators). Evolved from standalone analog to networked digital systems. 2 Risks: High impact if compromised (danger, disruption, cost). Integration with business networks increases attack surface. Often use unsupported OS (patching challenges). Lack of IT expertise in OT environments. Dependencies poorly understood (Alabama nuclear plant examples). 3 Threat Actors: Financial gain, Hacktivists, Nation-States (considered most serious threat). 4 SCADA Best Practices: Network protection strategy, Site surveys, Strong access control (logical/physical), Harden/patch/disable services, Review topology/connections, Unplug unnecessary connections, Pen test remaining connections. 5 Key Components: * PLC (Programmable Logic Controller): Automates/monitors processes. Secure physical access, change defaults, control network links. * Historian: Logs/stores all SCADA data for analysis and reporting. * Ladder Logic: Programming language/diagramming method for PLCs. * SIS (Safety Instrumented System): Monitors operations for safety, controls emergency stops. Target of Stuxnet (sent fake “OK” signals). 6 Stuxnet: Malware targeting specific Siemens PLCs (Iranian nuclear facility). Caused physical damage while hiding activity from SIS. Zero-day vulnerabilities still exist.

⠀C. OT Protocols

  • General: Rules for data transmission between industrial devices. Often lack built-in security.
  • Specific Protocols:
    • CAN bus: Vehicle networks (engine, windows). Shared broadcast, no encryption. Mitigation: Add encryption layer.
    • Modbus: Serial communication (PLCs). Widely used in SCADA. Limited addressing. Mitigation: Use Modbus TCP/IP variant with TLS.
    • DNP3: Electric/water utilities. Supports authentication/encryption. Layered protocol (App, Data Link). Can run over TCP/IP.
    • Zigbee: IoT wireless sensors. Uses symmetric keys. Risk: Sensitive data/key theft from node.
    • CIP (Common Industrial Protocol): Data organization/messaging. Can reject messages from untrusted devices if deployed correctly.
    • DDS (Data Distribution Service): Middleware protocol/API (air traffic control, autonomous vehicles). Peer-to-peer. Includes auth, access control, confidentiality, integrity features.

⠀D. Sector-Specific Considerations

  • Tailor cybersecurity programs based on industry-specific threats.
  • Energy: Complex environments, targeted by state actors.
  • Manufacturing: Financial/reputational damage risk.
  • Healthcare: Life-risk impact, data privacy (HIPAA), compliance burden.
  • Public Utilities/Services: Critical infrastructure, aging legacy systems, growing attack surface. Need proactive defense.
  • Financial Services: Heavily targeted, strict regulations, need advanced threat protection.

⠀E. Digital Forensics (Mobile/Collected Data Context)

  • Process: Identification -> Preservation (Hashing - MD5/SHA256) -> Collection (Chain of Custody critical) -> Examination -> Analysis -> Presentation -> Decision.
  • Scope: Applies to compromised systems, smartphones, tablets, cameras, USB drives.
  • Chain of Custody: Document WHO collected/possessed, WHERE collected/stored, HOW collected/stored, WHY accessed/moved. Essential for legal admissibility.
  • E-discovery: Finding/extracting/reviewing electronic info (structured/unstructured - emails, chats) for legal cases. Governed by data retention policies.

⠀Exam Essentials Summary:

  • Emerging Tech Impact: Understand the security/privacy implications of AI/ML/Deep Learning (Deep Fakes, NLP), Quantum Computing (crypto threat), Blockchain (immutable ledger), Homomorphic Encryption (compute on encrypted data), Big Data, VR/AR (privacy/safety), 3D Printing (IP theft/sabotage), Passwordless Auth, Nanotech, and Biometrics (accuracy - FRR/FAR/CER, data management).
  • Mobile Security Configurations: Know how to secure enterprise mobility using MDM/EMM, Managed Configs, App Control (Wrapping, MAM), Strong Auth (Password Policy, MFA, Tokens), Updates (Patch Repo, OTA), Remote Wipe, Wireless (WPA3), Certificates (SCEP risks), Profiles/Payloads, restricting short-range wireless (Bluetooth, NFC) & Peripherals, Location Controls (Geofencing/tagging risks), FDE, Tethering controls, Airplane Mode use, and DNS security (DoH).
  • Mobile Deployment Models: Differentiate COBO, COPE, CYOD, BYOD and their associated risks/benefits.
  • Mobile Security Considerations: Recognize threats like malware, botnets, smishing, clickjacking, and risks like unauthorized activation, recon via location data, data theft, health privacy concerns with wearables, unauthorized app stores, jailbreaking/rooting, sideloading, supply chain issues, and eFuse anti-tampering.
  • OT/ICS/SCADA Security: Understand the unique risks of connecting industrial control systems (PLCs, SIS, Historian) and embedded devices/IoT to networks. Know key protocols (CAN bus, Modbus, DNP3, Zigbee, CIP, DDS) and their security postures. Recognize sector-specific threat landscapes (Energy, Healthcare, Finance, etc.). Apply SCADA best practices (segmentation, hardening, access control, pen testing). Understand SoC, ASIC, FPGA concepts.
  • Forensics (Mobile): Apply core forensic principles (Chain of Custody, Order of Volatility - though less emphasized here, Preservation via Hashing) to mobile device investigations. Understand E-discovery concepts.
